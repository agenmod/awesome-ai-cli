# 📊 AI 兼容度评分标准

我们用 **5 维评分体系** 评估每个 CLI 工具对 AI Agent 的友好程度。

这个评分的目的不是制造排名焦虑，而是给工具开发者一个明确的方向：**"我的 CLI 对 AI 友好吗？还能从哪些方面改进？"**

## 五个评估维度

### 1. AI 设计 (AI Design)

> 这个工具是否从设计之初就考虑了 AI Agent 的使用场景？

| 分数 | 标准 |
|:----:|------|
| 5 | 文档/README 中明确提到 AI Agent，自带 Skills 和 MCP 支持 |
| 4 | 有明确的自动化/脚本化设计意图，但未显式提到 AI |
| 3 | 设计上偏向自动化，部分功能对 AI 友好 |
| 2 | 主要为人类设计，AI 可以勉强使用 |
| 1 | 完全面向人类的交互式设计 |

### 2. 结构化输出 (Structured Output)

> AI 能否直接解析工具的输出？

| 分数 | 标准 |
|:----:|------|
| 5 | 所有命令默认 JSON 输出，错误信息也 JSON 化，支持 field masks |
| 4 | 支持 `--output json` 或类似参数，大部分命令覆盖 |
| 3 | 部分命令支持结构化输出 |
| 2 | 输出格式有一定规律，AI 可勉强解析 |
| 1 | 纯人类可读文本，无结构化选项 |

### 3. 自查能力 (Self-Discovery)

> AI 能否自主发现这个工具有哪些命令、需要什么参数？

| 分数 | 标准 |
|:----:|------|
| 5 | 支持 `list-commands`，每个子命令 `--help` 详尽，参数有类型描述 |
| 4 | `--help` 输出详尽，覆盖所有子命令 |
| 3 | 有 `--help`，信息基本够用 |
| 2 | `--help` 输出简略，需要额外查文档 |
| 1 | 无帮助信息或极其简略 |

### 4. 预览模式 (Preview / Dry-run)

> AI 执行前能否看到将要发生什么？

| 分数 | 标准 |
|:----:|------|
| 5 | `--dry-run` 覆盖所有写操作，输出明确显示将执行的变更 |
| 4 | 大部分写操作支持预览 |
| 3 | 部分命令支持 `--dry-run` 或确认机制 |
| 2 | 有确认提示但无预览 |
| 1 | 执行即生效，无预览无确认 |

### 5. 上下文友好 (Context-Friendly)

> 这个工具是否尊重 AI 有限的上下文窗口？

| 分数 | 标准 |
|:----:|------|
| 5 | Skills 文件 < 2KB，输出支持 field masks / 分页，精确控制返回量 |
| 4 | Skills 文件合理，输出有分页或 limit 参数 |
| 3 | 文档大小适中，输出量偶尔过大 |
| 2 | 文档偏大，输出经常超长 |
| 1 | 巨大的 Skills 文件，输出无控制机制 |

## 评分汇总

| 总分范围 | 星级 | 说明 |
|:--------:|:----:|------|
| 24-25 | ⭐⭐⭐⭐⭐ | AI 原生设计，开箱即用 |
| 18-23 | ⭐⭐⭐⭐ | 高度适配，略有改进空间 |
| 12-17 | ⭐⭐⭐ | 基本可用，部分场景需适配 |
| 6-11 | ⭐⭐ | 有限支持，较多人工干预 |
| 1-5 | ⭐ | 基本不适配 AI 使用 |

## 标杆工具分析

### Google Workspace CLI — 满分标杆 (25/25)

- **AI 设计 5/5**：README 首段就写"designed for AI agents"，内置 MCP 服务
- **结构化 5/5**：所有输出 JSON，支持 field masks 精确控制返回字段
- **自查 5/5**：完整的命令树自描述
- **预览 5/5**：写操作均支持 dry-run
- **上下文 5/5**：Skills 文件平均仅 1.6KB，堪称教科书

### 飞书 Lark CLI — 近满分 (24/25)

- **AI 设计 5/5**：200+ 命令覆盖 11 个业务领域，MCP + Skills 全配
- **结构化 5/5**：全 JSON 输出
- **自查 5/5**：每个子命令可自描述
- **预览 5/5**：支持 dry-run
- **上下文 4/5**：Skills 文件较 Google 稍大，但仍在合理范围

### ffmpeg — 经典工具参考 (12/25)

- **AI 设计 2/5**：90 年代设计，完全面向人类
- **结构化 3/5**：`-print_format json` 可输出 JSON，但非默认
- **自查 3/5**：`-help` 完整但信息量巨大
- **预览 2/5**：无 dry-run，但 AI 训练数据充足可预判
- **上下文 2/5**：无 Skills，但 AI 已通过训练数据掌握

::: info 经典工具的特殊优势
ffmpeg、jq、curl 这类经典工具虽然 AI 兼容度评分不高，但 AI 在训练时已见过海量用法。它们不需要 Skills 说明书，AI 天然就会用。新工具则完全依赖显式说明。
:::

## 给工具开发者的建议

如果你正在开发一个 CLI 工具，想让 AI Agent 能顺利使用它：

1. **加一个 `--output json` 参数**，这是最低成本最高收益的改进
2. **消除所有交互式弹窗**，改用参数传入，加一个 `--no-interactive` 开关
3. **写一个精简的 Skills 文件**，控制在 2KB 以内，告诉 AI 核心命令和典型场景
4. **写操作加 `--dry-run`**，让 AI 执行前能预览
5. **大列表输出加 `--limit` 和 `--fields`**，避免淹没 AI 的上下文
