# ❓ FAQ

## CLI 相关

### CLI 和 GUI 有什么区别？

GUI（图形界面）是打开 App，看到按钮菜单，鼠标点来点去。CLI（命令行）是打开终端，敲一行文字，按回车，事情做完了。

打个比方：GUI 是去餐厅看菜单，指给服务员"我要这个"。CLI 是直接对厨房喊"宫保鸡丁，少油，多辣"。

### 为什么 2026 年大家突然做 CLI？

不是人类复古爱命令行，是 AI Agent 天生只懂文本交互。GUI 是给眼睛看的，AI 没有眼睛。CLI 是纯文字的，AI 天生就在这个世界里运作。

### CLI 和 MCP 什么关系？

MCP（Model Context Protocol）是 AI 和外部服务之间的标准通信协议，类似 AI 世界的 USB 接口。新一代 CLI 工具很多内置了 MCP 服务，一条命令就能启动。CLI 是执行层，MCP 是通信协议，两者互补。

### CLI 和 AI Plugin 什么关系？

一个新一代 CLI 工具 = 执行能力 + 通信协议（MCP）+ 使用说明（Skills），本质就是一个 Plugin。但 CLI 比 Plugin 更自由：跨平台（不锁定 AI 平台）、免审核（npm 发布即上线）、人和 AI 都能用、支持管道组合。

## 评分相关

### 评分依据是什么？

我们从 5 个维度评分：AI 设计、结构化输出、自查能力、预览模式、上下文友好。每项 1-5 分，总分 25 分。详见 [评分标准](/scoring)。

### 经典工具评分低，是不是不好用？

不是。ffmpeg、jq、curl 这些经典工具评分不高，但 AI 在训练时已见过海量用法。它们不需要 Skills 说明书，AI 天然就会用。评分主要反映"AI 原生设计程度"，经典工具有训练数据的隐性优势。

### 我不同意某个评分怎么办？

欢迎提 Issue 讨论或直接提 PR 修改，附上你的评分依据和测试环境。

## 使用相关

### 我不是程序员，能用吗？

可以。如果你用 Cursor、Claude Code 等 AI 编程工具，只需要安装 CLI 工具，剩下的交给 AI。你只需要用自然语言描述你想做什么。

### 安装时遇到权限问题怎么办？

参考 [快速安装](/install) 页面的常见问题部分，包含了 npm 权限修复和 brew 中国镜像等解决方案。

### 工具需要登录怎么办？

大多数工具首次使用需要认证。通常运行工具后会提示登录方式。目前各工具认证方式不统一，这是行业待解决的问题之一。

## 项目相关

### 如何提交新工具？

参考 [贡献指南](/contributing)。Fork 仓库，在 `cli-data/tools.json` 添加工具信息和 5 维评分，提交 PR。

### 这个项目的目标是什么？

成为 AI 时代 CLI 工具的发现和评估平台。帮助开发者快速找到适合 AI Agent 使用的 CLI 工具，同时推动工具开发者关注 AI 兼容性。
